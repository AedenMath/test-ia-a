## How to Contribute

We welcome contributions from everyone! Here's how you can help:

### Ways to Contribute

1. **Report Bugs**: Found an issue? Please open a GitHub issue with details about the problem.
2. **Suggest Features**: Have an idea? Open a feature request in the issues section.
3. **Submit Code**: Fix bugs or add features by submitting a pull request.
4. **Improve Documentation**: Help us improve our docs and README.
5. **Share Feedback**: Let us know what you think about the project.

### Contribution Guidelines

1. Fork the repository
2. Create a feature branch (`git checkout -b feature/your-feature`)
3. Make your changes and commit them (`git commit -m 'Add your feature'`)
4. Push to the branch (`git push origin feature/your-feature`)
5. Open a Pull Request with a clear description of your changes

## Recognition Policy

Contributors will be recognized based on their contributions:

- **Code Contributions**: Bug fixes, features, and code improvements
- **Documentation**: Documentation updates, examples, and guides
- **Community**: Helping others, answering questions, and community building
- **Testing**: Adding tests and improving test coverage
- **Maintenance**: Reviewing PRs, triaging issues, and project maintenance
